GESTIÓN DE LA CALIDAD DEL SOFTWARE

Fundamentos

La gestión de calidad de software busca asegurar que un programa cumpla con las necesidades del usuario, sea confiable, fácil de mantener y funcione correctamente. Su objetivo es prevenir errores en lugar de corregirlos después, reduciendo costos y mejorando la satisfacción del cliente. Se apoya en buenas prácticas de desarrollo, revisiones, pruebas y uso de estándares internacionales.

Conceptos clave

- Calidad del software: grado en que el producto cumple con los requisitos esperados (funcionalidad, seguridad, usabilidad, etc.). - Confiabilidad: que el software funcione sin fallos durante un tiempo y condiciones específicas. - Mantenibilidad: facilidad para corregir errores, actualizar o mejorar el software. - Eficiencia: buen uso de recursos como memoria, tiempo de ejecución y energía. - Usabilidad: que el usuario lo entienda y use sin dificultad. - Portabilidad: que el software pueda adaptarse a diferentes plataformas o sistemas.

Métricas

Las métricas son medidas que ayudan a evaluar la calidad del software. Ejemplos:  Densidad de defectos: número de errores encontrados por cada mil líneas de código.  Cobertura de pruebas: porcentaje de código revisado mediante pruebas.  Tiempo medio de fallo (MTTF): cuánto tiempo pasa hasta que aparece un error.  Complejidad ciclomática: mide la dificultad de comprender y mantener el código.  Satisfacción del usuario: encuestas y retroalimentación que reflejan la experiencia real.

Estándares

Son guías reconocidas internacionalmente que aseguran que el software siga buenas prácticas:  ISO/IEC 25010: modelo que define características de calidad del software (funcionalidad, usabilidad, seguridad, etc.).  ISO 9001: estándar general de gestión de calidad aplicable también al desarrollo de software.  CMMI (Capability Maturity Model Integration): modelo mmque evalúa la madurez de los procesos de desarrollo.  IEEE 829: estándar para la documentación de pruebas de software
