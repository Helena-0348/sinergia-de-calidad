Glosario de términos de desarrollo de software (A–Z)

Automatización de pruebas: Uso de herramientas que permiten ejecutar pruebas sin intervención manual, ahorrando tiempo y reduciendo errores. Bug / Defecto: Error en el software que provoca un funcionamiento incorrecto. Calidad del software: Grado en que un programa cumple con los requisitos esperados (funcionalidad, seguridad, usabilidad, etc.). CMMI (Capability Maturity Model Integration): Modelo que mide el nivel de madurez y optimización de los procesos de desarrollo de software. Confiabilidad: Capacidad del software para funcionar sin fallos en un periodo y condiciones específicas. Eficiencia: Uso óptimo de recursos como memoria, tiempo de ejecución y energía. Entrega incremental: Estrategia de liberar el software en partes pequeñas y utilizables. Estándar: Conjunto de reglas internacionales que guían el desarrollo y las pruebas de software. ISO/IEC 25010: Estándar internacional que define características de calidad del software (usabilidad, eficiencia, confiabilidad, etc.). Iteración: Ciclo repetido de desarrollo y mejora de un producto. Kanban: Sistema visual para gestionar tareas en un tablero con columnas (pendiente, en progreso, terminado). Mantenibilidad: Facilidad para corregir errores, actualizar o mejorar el software. Métrica: Dato medible que permite evaluar la calidad o el rendimiento del software. Modelo en Cascada: Método lineal de desarrollo donde cada etapa inicia al finalizar la anterior. Modelo en V: Similar al modelo en cascada, pero con pruebas asociadas a cada etapa de desarrollo. Modelo Iterativo/Incremental: Construcción de software en versiones parciales que mejoran en ciclos. Portabilidad: Capacidad del software de adaptarse a diferentes dispositivos o sistemas. Prototipo: Versión inicial y simple de un software que sirve para pruebas rápidas. Prueba de Aceptación: Validación realizada por clientes o usuarios finales para confirmar que cumple lo esperado. Prueba de Integración: Verificación de cómo interactúan distintos módulos del sistema. Prueba de Sistema: Evaluación del software completo en su entorno real. Prueba Unitaria: Verificación de una parte pequeña del código, como funciones o módulos. QA (Quality Assurance): Actividades para asegurar la calidad del software durante todo el proceso de desarrollo. QC (Quality Control): Verificación final de que el producto cumple con lo esperado. Requisitos: Necesidades o condiciones que el software debe cumplir. Scrum: Metodología ágil basada en roles, sprints y reuniones cortas para entregar mejoras continuas. SDLC (Software Development Life Cycle): Conjunto de etapas que sigue un software desde la idea hasta su retiro (requisitos, diseño, desarrollo, pruebas, implementación, mantenimiento). Usabilidad: Qué tan fácil es para el usuario aprender y usar el software.
